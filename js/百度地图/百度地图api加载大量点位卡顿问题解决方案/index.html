百度地图api加载大量点位卡顿问题解决方案

发表于 2020-06-10 更新于 2020-12-07 分类于 js ， 百度地图 阅读次数：

背景

在使用百度地图进行点位标注时，很多人喜欢用Market进行标注。
Marker 此类表示地图上一个图像标注。 构造函数 Marker(point: Point , opts: MarkerOptions ) 创建一个图像标注实例。point参数指定了图像标注所在的地理位置

因为Market类提供的方法多，如可换标注图标等等，是常用的百度地图api类。但该类有一个非常大的缺陷，当点位超过1000个之后，会造成加载时间过长、地图拖动卡顿等现象，无法使用。

解决方案

技术一：加载海量点

参考：http://lbsyun.baidu.com/jsdemo.htm#c1_19

这时点位加载速度会提升很大，2000个点秒加载，而使用Markers则在5秒以上才能加载完成。

如果需要在点位上加文本标示，如：http://lbsyun.baidu.com/jsdemo.htm#c1_14
使用上面链接demo中的方法，循环加载即可，但又会造成加载缓慢，而且页面卡顿的情况，所以要用到下面的技术。

技术二：动态加载label

  • 整体思路：地图添加拖拽和放缩的响应方法，当地图界面范围发生变化时，获取当前地图四个角落点的坐标值，判断在此坐标范围内的点，如超过500个（500个以内的Label添加在一秒以内，在多会影响速度，也没有意义，因为Label会叠加在一起，完全看不清楚），则不显示Label，如果在此坐标范围内的点不超过500个，则显示label，地图范围外的label不显示。这样可以最大程度的减少地图上的label数量，可以使地图流畅拖拽。注意，加载时间绝大部分是发生在向地图上添加label的这个动作，所以要分批添加点位，每次500个左右，另外，如果添加后随即隐藏，也可以减少一半以上的加载时间。
